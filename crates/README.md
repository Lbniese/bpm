# crates/

Reserved for the multi-crate workspace layout described in
(`bpm-cli`, `bpm-core`, `bpm-manifest`, `bpm-lockfile`, `bpm-resolver`,
`bpm-registry`, `bpm-store`, `bpm-graph`, `bpm-materializer`, `bpm-lifecycle`,
`bpm-bench`).

The current implementation intentionally stays a single crate (`bpm`, at the
("do not add placeholder crates or abstractions"). This directory exists so
the intended future layout is discoverable; it is not used yet.
